# Architecture — NLACP-AttributeExtractor

## Package Layout

```
nlacp/
├── pipeline/
│   └── pipeline.py          ← orchestrates all 6 modules
├── extraction/
│   ├── relation_candidate.py ← subject/action/object + attr extraction (Module 1)
│   ├── env_extractor.py      ← temporal/spatial env attributes (Module 1b)
│   └── short_name_suggester.py ← short name generation (Module 2)
├── normalization/
│   ├── category_identifier.py  ← subject/object/env classification (Module 4)
│   ├── namespace_assigner.py   ← ABAC namespace routing (Module 3)
│   └── data_type_infer.py      ← NER-based type inference (Module 5)
├── mining/
│   ├── attribute_cluster.py    ← DBSCAN clustering (Module 2b)
│   └── namespace_hierarchy.py  ← inheritance hierarchy (Module 3b)
├── evaluation/
│   └── evaluator.py            ← P/R/F1 computation
└── io/
    └── dataset_builder.py      ← JSON policy dataset read/write
```

## Main Pipeline Flow

```
Input sentence
    ↓
[Module 1] relation_candidate.py   → subject, action, object, raw attrs
           env_extractor.py        → temporal/spatial env attrs
    ↓
[Module 2] short_name_suggester.py → short_name for each attr
    ↓
[Module 4] category_identifier.py  → category (subject/object/environment)
                                     ← must run BEFORE namespace!
    ↓
[Module 3] namespace_assigner.py   → hierarchical namespace string
                                     (subject:role:X, environment:time:Y, ...)
    ↓
[Module 5] data_type_infer.py      → data_type (string/datetime/integer/boolean)
    ↓
[Module 6] dataset_builder.py      → persist to outputs/policies/policy_dataset.json
    ↓
Output: ABAC policy dict
```

## Key Design Decisions

| Decision | Rationale |
|---|---|
| Module 4 before Module 3 | Category is source of truth; namespace must READ category, not set it |
| `env_extractor` as separate module | Env attrs have different extraction logic (rule + NER hybrid) |
| `outputs/` separate from `src/` | Generated artifacts should not live next to source code |
| `experiments/` separate | CNN, GloVe clustering are not in main pipeline yet |
| `legacy/` separate | LitroACP is a different sub-project |

## Core vs Experiments vs Legacy

| Layer | Description |
|---|---|
| **Core** (`nlacp/`) | Main pipeline, importable library, always maintained |
| **Scripts** (`scripts/`) | CLI entry points, run/eval/annotate tools |
| **Tests** (`tests/`) | Integration and unit tests |
| **Experiments** (`experiments/`) | CNN classifier, GloVe clustering — not in main path |
| **Legacy** (`legacy/`) | LitroACP sub-project, old research notes |
| **Data** (`data/`) | Raw corpora, gold annotations — never modified by pipeline |
| **Outputs** (`outputs/`) | Generated JSON artifacts — regenerated by pipeline |
